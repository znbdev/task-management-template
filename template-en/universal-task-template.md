## 💡 Usage Guide

### Best Practices for Effort Estimation

1. **Granularity Control**: Individual task estimates should **not exceed 4 hours (0.5 day)**. The finer the breakdown, the more accurate the estimate.
2. **Historical Data Reference**: Refer to actual effort from similar past tasks to avoid being overly optimistic or pessimistic.
3. **Context Switching Consideration**: If handling multiple tasks simultaneously, add 10-15% time buffer.

### Variance Analysis & Retrospective

* **Variance Formula**: $\frac{\text{Actual} - \text{Estimated}}{\text{Estimated}} \times 100\%$
* **Variance < -20%**: Estimate was too conservative. Can be compressed next time.
* **Variance > 20%**: Estimate was too optimistic. Need to analyze reasons (requirements change? technical challenges? external blockers?)
* **Regular Retrospective**: After completing 3-5 tasks, review variance patterns and continuously optimize estimation ability.

### Status Transition Recommendations

```
🟩 Not Started → 🟨 In Progress → 🟦 Blocked (if applicable) → ⬜ Completed
                                  ↓
                              ❌ Cancelled (if task is no longer needed)
```

### Adjustment Suggestions for Different Task Types

* **Feature Development**: Focus on breaking down the "Implementation & Development" phase
* **Bug Fix**: Add "Problem Identification" and "Regression Testing" steps
* **Research Analysis**: Strengthen the "Research & Design" phase, reduce "Implementation & Development"
* **Documentation**: Simplify technical steps, add "Review & Revision" steps
* **Technical Optimization**: Add "Performance Testing" and "Comparison Validation" steps
